Developer Notes

About This Spotify Clone

This project is intentionally built with a vanilla frontend stack: HTML, CSS, and JavaScript. The goal was to create a lightweight music player with clean interaction design, responsive behavior, and maintainable code without relying on heavy UI frameworks.

Developer Profile

I enjoy building practical frontend interfaces where visual polish and interaction quality matter equally. My focus areas are UI architecture, responsive systems, and state-driven behavior for media-rich web experiences.

GitHub LinkedIn

Why This Build Exists

Building a streaming-style UI looks simple at first, but real complexity appears when playback state, control synchronization, keyboard shortcuts, and responsive behavior must all work together. This build is designed as a practical frontend engineering exercise, not only a visual clone.

Implemented Features

  • Master play/pause, previous/next controls, and per-song play buttons
  • Progress seekbar with current and total duration labels
  • Shuffle and repeat modes with proper state indication
  • Volume slider, mute toggle, and keyboard shortcuts
  • Search filter for playlist and active-song highlight
  • Responsive UI for desktop, tablet, and mobile screens

Technical Approach

State management is intentionally minimal. Core states such as current track index, playback status, shuffle, and repeat mode are stored in JavaScript. UI updates are centralized through helper functions so that control states and track states remain consistent.

On the styling side, the layout uses a flex and grid combination with a strong visual hierarchy. Gradients, contrast cards, and spacing rhythm are used to improve readability and control discoverability, not only visual decoration.

What I Would Improve Next

  • Local storage persistence for last song and playback position
  • Favorites playlist and history queue
  • Mini-player mode for compact screens
  • Accessibility pass for high contrast and full keyboard navigation
  • Audio visualization panel for now-playing track

Built with focus on clean interaction, maintainable code, and practical frontend behavior.
